# MetaPlogging 전체 현황 대시보드

MetaPlogging DB의 실제 데이터를 읽기 전용으로 가져와 표시하는 수동 갱신형 대시보드입니다. 대시보드 접속 로그인은 없습니다.

## 수동 갱신

프로젝트 폴더에서 다음 명령을 실행합니다.

```bash
python3 sync_db.py
```

터미널에 `DB 비밀번호:`가 표시되면 기존 DB 접속 비밀번호를 입력합니다. 입력한 글자는 화면에 나타나지 않는 것이 정상입니다. 비밀번호는 코드나 생성 파일에 저장되지 않습니다.

갱신이 완료되면 `db-snapshot.js`가 새로 생성됩니다. 실행 중인 대시보드 브라우저를 새로고침하면 최신 스냅샷이 표시됩니다.

## 로컬 대시보드 실행

```bash
python3 -m http.server 4173 --bind 127.0.0.1
```

브라우저에서 `http://127.0.0.1:4173`을 엽니다.

`db-snapshot.js`에는 내부 세션과 위치 데이터가 포함되므로 GitHub에는 업로드되지 않도록 설정되어 있습니다. 이 수동 방식은 현재 컴퓨터의 내부 모니터링 화면에서 사용합니다.

## 데이터 범위

- 전체 사용자·세션·이동 거리·활동 시간·사진 수
- 시간대별·월별 사진 수집량
- 사용자별 익명 수집량·이동 거리
- 사진 좌표 히트맵
- GPS 좌표·사진 누락 및 미완료 세션
- 실제 GPS 포인트 기반 속도 노이즈

DB는 `SELECT`만 사용하는 읽기 전용 세션으로 조회합니다. 이 방식은 실행 시점의 실제 데이터 스냅샷이며 실시간 자동 갱신은 아닙니다.
