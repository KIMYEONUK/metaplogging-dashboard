# MetaPlogging 전체 현황 대시보드

MetaPlogging DB의 실제 데이터를 읽기 전용으로 가져와 표시하는 수동 갱신형 대시보드입니다. 대시보드 접속 로그인은 없습니다.

## 수동 갱신

프로젝트 폴더에서 다음 명령을 실행합니다.

```bash
python3 sync_db.py
```

터미널에 `DB 비밀번호:`가 표시되면 기존 DB 접속 비밀번호를 입력합니다. 입력한 글자는 화면에 나타나지 않는 것이 정상입니다. 비밀번호는 코드나 생성 파일에 저장되지 않습니다.

갱신이 완료되면 `db-snapshot.js`가 새로 생성됩니다. 플로깅 세션의 대표 GPS 좌표를 Nominatim으로 시·군·구로 변환하고, 결과는 `geocode-cache.json`에 저장해 같은 좌표를 반복 요청하지 않습니다. 사진 EXIF 좌표는 세션 GPS가 전혀 없을 때만 보조로 사용합니다. 대시보드의 새로고침 버튼은 마지막으로 게시된 스냅샷을 캐시 없이 다시 불러옵니다.

## 로컬 대시보드 실행

```bash
python3 -m http.server 4173 --bind 127.0.0.1
```

브라우저에서 `http://127.0.0.1:4173`을 엽니다.

`db-snapshot.js`에는 실명·이메일이 포함되지 않습니다. 사용자와 세션 ID는 익명 번호로 바뀌고, 위치 좌표는 약 100m 단위로 낮춰 공개 대시보드에 사용합니다.

갱신 결과를 공개 페이지에도 반영하려면 다음 명령을 실행합니다.

```bash
git add db-snapshot.js
git commit -m "Update anonymized DB snapshot"
git push
```

## 데이터 범위

- 전체 사용자·세션·이동 거리·활동 시간·사진 수
- 시간대별·월별 사진 수집량
- DB 장소 정보 및 캐시된 Nominatim 결과 기반 시·군·구 수거 비율
- 사용자별 익명 수집량·이동 거리
- 플로깅 세션 대표 GPS 기반 히트맵
- GPS 좌표·사진 누락 및 미완료 세션
- 비공개 원본 GPS 포인트에서 계산한 속도 노이즈(공개 파일에는 이동 경로 제외)

DB는 `SELECT`만 사용하는 읽기 전용 세션으로 조회합니다. 이 방식은 실행 시점의 실제 데이터 스냅샷이며 실시간 자동 갱신은 아닙니다.
