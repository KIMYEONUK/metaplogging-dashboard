# MetaPlogging API 연동 대시보드 (별도 작업본)

원본 GitHub 저장소를 수정하지 않고 만든 독립 로컬 작업본입니다.

## 실행

브라우저의 파일 열기가 아니라 HTTP 서버로 실행해야 합니다.

```bash
cd metaplogging-dashboard-integrated
python3 -m http.server 4173 --bind 127.0.0.1
```

그다음 `http://127.0.0.1:4173`을 열고 MetaPlogging 계정으로 로그인합니다. 비밀번호는 저장하지 않으며, 발급된 토큰은 브라우저 탭 세션 저장소에만 보관합니다.

API 주소는 `config.js` 한 곳에서 관리합니다.

## 현재 연동 범위

- 플랫폼 전체: 가입자, 누적 거리, 누적 수거량, 사용자 리더보드
- 로그인 계정: 세션, 쓰레기 지점, 사진, GPS 좌표
- 클라이언트 계산: 시간·월별 추이, 좌표/사진 누락, 미완료 세션, 20km/h 초과 GPS 노이즈, 히트맵
- 인증: 로그인, 액세스 토큰 자동 갱신, 로그아웃

## 서버 측 선행 조건

공개 GitHub Pages는 HTTPS이지만 현재 API는 HTTP이므로 브라우저가 직접 연결을 차단합니다. 운영 배포에는 다음 중 하나가 필요합니다.

1. API 서버에 HTTPS 적용 후 `config.js`를 HTTPS 주소로 변경
2. 대시보드와 같은 HTTPS 출처의 리버스 프록시에서 `/api`를 백엔드로 전달

또한 현재 세션 API는 본인 데이터만 반환합니다. 관리자 대시보드에서 전 사용자 품질·기간·지역 통계를 정확히 보려면 역할 기반 관리자 인증과 서버 집계 API가 추가로 필요합니다. 필요한 응답은 최소한 아래 범위를 포함해야 합니다.

- 기간별 수집량 집계
- 전 사용자 세션/사진/좌표 품질 집계 및 오류 목록
- 날짜·사용자별 쓰레기 지점 좌표
- 전체 활동 시간

관리자 API가 생기면 `data.js`의 `loadDashboardData()`만 해당 엔드포인트에 맞춰 교체하면 됩니다.
